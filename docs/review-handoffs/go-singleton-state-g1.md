# Singleton-state topology G1 builder handoff

Status: APPROVED after fresh adversarial review and targeted remediation
recheck. The coordinator has not yet committed this parcel.

## Intent

- Replace logical slug/group state topology with the v2 invariant that every
  generated resource type is one state unit: label equals type and members is
  the one-element list containing that type.
- Remove deployment inputs `strategy`, `groups`, and `bind_references`, and
  pack-registry input `slug_group`, with deterministic fail-closed diagnostics
  pointing to `docs/singleton-state-topology-v2.md`.
- Emit the new schema-v2 root catalog without `slug_label` while retaining the
  immutable v1 catalog for the frozen Node rollback/oracle lane.
- Replace only the topology-dependent v1 differential cases with explicit Go
  v2 authority tests. Retain topology-independent Node differentials and the
  C4/D5 behavior, artifact, transcript, and exit-code comparisons after their
  fixtures are expressed as singleton topology.
- Preserve the transitional G1 reference default: omitted or false
  `cross_state_references` remains disabled. G2, not this parcel, promotes the
  omitted case to cross-state and adds the cycle gate.

## First adversarial review remediation

The fresh review returned **Request changes** on three verified findings; no
catalog, parser, singleton, consumer-isolation, C4/D5, or oracle-digest defect
was found.

1. The `dist/iw` prerequisite list omitted compiled Darwin `.s` files. Root
   cause: an extension allow-list. Fix: depend conservatively on every file
   below `go/`, create `dist/` in the build recipe, and prove an assembly
   change schedules the build.
2. Generation/topology replacement gates were weaker than the retired lane.
   Root cause: treating all generation as topology-dependent. Fix: restore
   exact frozen-Node module and implicit-singleton gen-env comparisons; add
   complete committed v2 roots/scope/plan/gen-env bytes and trees; make the
   advertised differential target execute them.
3. New Go-v2 tests assumed ignored `dist/` already existed. Fix: provision the
   runtime directory only when absent, remove only the test binary and an
   empty test-owned directory, and pass the tests with `dist/` absent.
