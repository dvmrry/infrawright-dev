## Initial Adversarial Findings and Remediation

- Initial verdict: request changes with four blockers.
- Foreign clean imports/root content: remediated with an exact top-level
  workspace allowlist plus a synchronous guard inside the saved-plan evidence
  transaction. The guard rejects foreign address/type/provider/import ID,
  previous-address moves, output changes, resource drift, missing staged
  imports, and imports outside the state-aware delta.
- Rejected evidence retention: remediated by invalidating prior outputs before
  collection and treating every later output as one cleanup transaction.
  Failed plan, stale fingerprint, hook rejection, assessment failure,
  persistence failure, dirty/scope rejection, and early-rerun failure now
  remove `tfplan`, `tfplan.sources`, the assessment output, the pending plan,
  and staged imports. Cleanup attempts all paths before reporting a secondary
  cleanup error.
- Permissive plan mode: remediated by writing Terraform's pending output inside
  a verified `0700` directory, binding it with no-follow regular/single-link
  checks, changing it to `0600`, synchronizing it, and atomically moving and
  rechecking the same inode at the final path. A pre-existing `0755` env root is
  tightened to `0700` in the regression test.
- Nested symlink escape: remediated by no-follow directory binding, canonical
  containment checks, exact content allowlists, no-follow runtime file opens,
  and post-write rechecks. Module root, env root, and intermediate-parent
  symlink probes all fail without touching the external target.
- The first review's nonblocking state-matrix risk was also narrowed with
  partial-delta, unknown-address, malformed-output, and failed-listing tests.
- Re-review must inspect only these remediated surfaces and their focused
  regressions; the credentialed live gates remain unresolved.

## Second Adversarial Findings and Remediation

- Second verdict: request changes with three filesystem blockers; the reviewer
  confirmed the foreign-plan guard and saved-plan permission blocker closed.
- Foreign-content rerun retention: workspace directory binding is now separate
  from content validation. Known plan outputs and staged imports are removed
  after safe authority binding but before exact-content validation, collection,
  or any other rerun phase. A successful-run/foreign-file/rerun regression
  proves all prior outputs are absent.
- Directory replacement: the URL-Categories layout now retains `dev`/`ino`
  identities for every root, artifact, module, environment, private, and plan
  directory. Every existing checkpoint and cleanup reopens each directory with
  `O_NOFOLLOW`, compares identity/mode/containment, and refuses path-based work
  after replacement. Env/module/config/private-plan replacement probes all fail
  without changing the external sentinel.
- Destructive hard-link rejection: runtime text files are no longer opened
  with `O_TRUNC`. Each write uses a new exclusive `0600` single-link file in the
  destination directory, synchronizes it, atomically replaces an existing
  regular workspace entry, and verifies the published inode. The external side
  of a pre-existing hard link retains byte-identical content.
- The second review's nonblocking synchronous-guard note is also closed:
  thenable plan-guard results are rejected just like asynchronous finalizers.
- Final re-review is limited to these three remediations. Both credentialed
