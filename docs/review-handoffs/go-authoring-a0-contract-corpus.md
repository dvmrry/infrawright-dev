## Invariants Claimed

- A verified input can qualify only when `sourcebind.LoadVerified` returned its
  opaque loader-owned state and `sourcebind.RequireQualification` accepted that
  exact state. A1 receives an opaque `QualifiedInputs` view; every analysis
  snapshot is a defensive deep copy, so caller mutation cannot change later
  snapshots. Contract renderers are codecs, not trust attestations.
- Every source-report resource key exactly equals the selected input resource
  set. Provider and SDK locations, module versions, package paths, and files
  bind back to the exact input manifest/observation.
- Every location explicitly selects the provider tree or one exact SDK module.
  Every call step separates its bound caller/callee declarations from its call
  site; the first caller exactly equals the row's Read callback and each later
  caller exactly equals the preceding callee. Resolved helper/SDK steps require
  a callee; raw HTTP and unresolved dispatch are terminal and have none.
- SDK imports resolve by slash-boundary longest module-prefix. The first SDK
  callee exactly equals `sdk_call`; overlapping or cross-SDK attribution fails
  closed. Endpoint locations exactly equal the terminal raw-HTTP call site.
- `observed_http` and endpoint-bearing ambiguous chains terminate in a
  location-identical `raw_http` request-construction step.
- `dynamic_method` and `dynamic_path` terminate in `raw_http`;
  `dynamic_dispatch` terminates in explicit unresolved dispatch.
- Package-scope and function-scoped provider registrations are both representable.
- OpenAPI is isolated: absent/unavailable input cannot remove source evidence;
  ambiguity candidates must be viable source-endpoint matches; conflict
  operations must positively differ.
- Verified rows project `legacy_mapped=true` only for `observed_http`.
- Local Git commands cannot run checkout-controlled fsmonitor/hooks, follow a
  checkout `core.worktree` redirect, prompt, page, or exceed bounded output.
- Local Go replacements are identity-checked against explicit SDK roots but
  never used as analyzer byte locators.
- Exact hashes and post-capture Git checks guard accepted bytes; callers must
  keep supplied source roots immutable for the duration of loading.
- All four public artifact decoders require bytes to equal canonical rendering,
  so downstream digests bind the accepted artifact bytes rather than a
  noncanonical equivalent that happens to re-render the same way.
- The ZPA matrix gate never emits or accepts endpoint qualification.

## Formal Review Loop

Two independent fresh-context reviews initially requested changes. The
accepted findings map to the corrected surface as follows:

- Create-only request accepted as Read-rooted -> explicit declaration-bound
  caller/callee edges -> Create-decoy and broken-adjacency regressions.
- SDK import attributed to an overlapping parent module -> one slash-boundary
  longest-prefix resolver and exact first-SDK-callee join -> parent/nested,
  sibling-prefix, wrong-entry, and cross-SDK regressions.
- Provider/SDK relative-path collision -> explicit provider/SDK location scope,
  exact SDK module, and symbol package path -> same-path/wrong-root regression.
- Digest of canonical reconstruction accepted for noncanonical artifact bytes
  -> strict decode/validate/render byte equality on every public decoder ->
  whitespace and key-order regressions for input provenance and source report.

The original review reports are recorded beside this handoff. Both original
reviewers rechecked the corrected staged surface and approved it with no
remaining blocking findings.

## Known Deferrals

- A1 AST/provider-to-SDK mapper implementation and its CLI integration.
- Actual OpenAPI parsing/resolution and adapter qualification.
- External ZPA checkout verification. Run the explicit test with a reviewed
  `ZPA_PROVIDER_SOURCE` root at the pinned `v4.4.6` commit before relying on the
  source-file/range recheck in a release ceremony.
- Remote/versioned Go `replace` directives are intentionally rejected by
  source-provenance-v1; only local replacement directories are modeled.
- Provider/SDK `repository` strings are reviewed manifest claims in A0. Local
  loading verifies revision, exact bytes, module identity, and replacements;
  it does not infer or compare a checkout remote URL.
- OpenAPI conflict `basis_reference` remains diagnostic-only and cannot change
  source classification. Before A3 emits conflicts, bind trusted/explicit
  basis values to concrete reviewed adapter inputs rather than free-form text.
- The parent process `PATH` is trusted to select the Git executable.
- Full descriptor-rooted traversal for every provider/SDK read is not added in
  A0. Exact file hashes, stable final-file identity, immutable-root precondition,
  and pre/post Git snapshots are the present boundary.
- Internal Artifactory revalidation for `golang.org/x/mod v0.29.0`.

## Review Focus

- Try to qualify zero/manual `VerifiedInputs`, snapshot zero/manual
  `QualifiedInputs`, or mutate one snapshot and influence a later snapshot.
  Plain contract artifacts must never mint the opaque qualified view.
- Try to splice a Create/Delete-only request beneath a Read row, break one
  caller/callee edge, collide provider and SDK filenames, or claim a broad SDK
  module when a nested module owns the import.
- Feed every decoder semantically equivalent whitespace/key-order variants and
  confirm no downstream digest join accepts them.
- Try to overclaim an endpoint without terminal raw HTTP construction, add or
  drop selected resources, bind an SDK call to another module/version/file, or
  leak an unverified path.
- Attack package-scope versus `Provider()`-scoped registration positions and
  the unresolved interface/Create-only decoy.
- Attack ambiguous-chain canonical ordering and all seven summary counts.
- Attack OpenAPI ambiguity/conflict with unrelated or same-endpoint operations;
  confirm it never changes source bytes/counts.
- Configure hostile repository `core.fsmonitor`, `core.worktree`, inherited
  `GIT_DIR`, output floods, dirty/untracked bound files, and mismatched local
  replacement roots.
- Independently reproduce every synthetic digest and the deterministic provider
  Git revision. Confirm no candidate mapper generated the expected corpus.
- Confirm the ZPA gate uses the frozen SHA for curated semantics rather than
  recreating the retired validator, and never labels anchors as endpoint proof.
- Challenge the accepted immutable-root/PATH threat boundary explicitly.
