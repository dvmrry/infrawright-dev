# Builder handoff — in-process post-render HCL formatting

## Intent

- Replace one `terraform fmt -` subprocess per generated `.tf` and
  `.tftest.hcl` file with a fail-closed in-process token-formatting pass.
- `modules generate` and `gen-env` should no longer require a Terraform binary
  or Terraform-supported execution platform merely to format generated files.
- Keep `internal/tfrender` as the byte-producing renderer. Do not construct HCL
  through `hclwrite` AST APIs, change generated bytes, or alter commands that
  actually execute Terraform.
- Keep `--terraform` accepted on the two generation commands for Node CLI
  compatibility. It is intentionally unused by their formatter.

## Expected Delta

- Expected behavior change: formatting occurs in-process after a successful HCL
  parse. `modules generate` and `gen-env` no longer spawn Terraform for each
  file and no longer enter the pre-dispatch Terraform platform gate.
- Expected report/count/coverage changes: none.
- Expected generated-output changes: none.
- Expected no-op areas: `tfrender`, JSON rendering, transform, topology,
  adopt/plan/apply, Terraform process bounds, evidence, and all provider logic.
- Error-path delta: malformed rendered HCL now returns a stable
  `generated HCL is invalid` diagnostic from the in-process parser instead of a
  Terraform subprocess error. Both paths fail before writing the invalid file.

## Review Focus

- Highest-risk files or paths: `go/internal/modulesgen/formatter.go`, the two
  full-corpus differential tests, and the `requiresTerraformExecution` change.
- Specific assumptions to attack:
  - Confirm every production HCL formatting call now uses `NewHCLFormatter` and
    no per-file `terraform fmt` production path remains.
  - Confirm `hclwrite.ParseConfig` is checked before `Format` and there is no
    fallback that writes bytes after diagnostics.
  - Confirm `.tftest.hcl` files are included in the 755-file module corpus.
  - Confirm only `gen-env` and `modules generate` left the Terraform platform
    gate, while every real Terraform invocation remains gated.
  - Confirm `--terraform` is accepted but cannot influence formatting.
  - Confirm the HCL dependency does not pull `go/packages` into the compiled
    program and is available in the enterprise mirror.
- Source evidence the reviewer should verify: Node formatter call sites,
  upstream HCL parse/format behavior, `needsTerraformFormat`, and all callers of
  `requiresTerraformExecution`.
- Generated artifacts the reviewer should compare: the 68 committed HCL
  goldens, full 755-file module corpus, 453-file environment corpus, and the
  standing Generation differential tree.
- Edge cases that could silently overclaim, remap, drop, or weaken evidence:
  malformed/partial HCL, comments and interpolation tokens, `.tftest.hcl`,
  Terraform/HCL formatter-version skew, missing Terraform on generation-only
  hosts, and an accidental platform-gate removal from a real execution path.
