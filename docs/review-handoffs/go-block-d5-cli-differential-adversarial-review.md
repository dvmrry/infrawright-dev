# Block D5 CLI and Differential Corpus — Adversarial Review

## Blocking Findings

### The CLI corpus did not exercise broad Apply overrides

- Finding: the first D5 candidate's incomplete-plan differential omitted
  `--allow-destroy` and `--allow-plan-changes`, and its composition unit did not
  assert the parsed Apply controls reached `ExactPlanApplyOptions`.
- Source evidence: the original arguments at
  `block_d5_differential_test.go:394-395` included only `--main-branch` and
  `--terraform`; the original unit invoked the three flags without inspecting
  the resulting option fields.
- Impact: D4 proved the domain gate, but D5 did not yet prove through the real
  CLI and frozen Node oracle that broad overrides cannot bypass
  `complete:false`.
- Required change: pass both broad flags through the frozen differential,
  preserve exact Node/Go exit and stderr, assert no fake Apply call, and lock
  down every supplied flag, main branch, and selector at the composition seam.
- Regression: the corrected differential now exercises both broad flags and
  compares the init/show transcript while rejecting any `arg=apply`; the unit
  asserts all three allow flags, main branch, and selectors.

### Reachable adoption diagnostic bytes diverged from Node

- Finding: Go's first diagnostic helper disabled HTML escaping but still
  escaped U+2028/U+2029 and rendered `json.Number` as a bare token rather than
  lossless-json's enumerable `LosslessNumber` object.
- Source evidence: Node 24.15 `JSON.stringify` emits literal U+2028/U+2029 and
  losslessly parsed numbers as
  `{"isLosslessNumber":true,"value":"<lexeme>"}`. Skipped and unsupported
  labels reach this helper before identity validation.
- Impact: frozen classification stderr differed for reachable scalar inputs.
- Required change: recursively map `json.Number` to the Node object shape,
  restore only genuinely unescaped line-separator escapes to literal UTF-8,
  and add frozen vectors for unsafe/exponential numbers and U+2028/U+2029.
- Regression: the corrected frozen classification corpus covers both line
  separators, an unsafe integer, an exponential number, HTML-sensitive text,
  and an absent label; complete stderr is byte-compared.

## Non-Blocking Risks

### Arbitrary object-valued labels retain Go map key order

- Risk: arbitrary object-valued skipped labels can render properties in Go's
  sorted map order instead of JavaScript insertion order.
- Source evidence: the established lossless Go decoder materializes JSON
  objects as `map[string]any`, and `SnakeJSONKeys` already documents that the
  original insertion order is unrecoverable.
- Why it is non-blocking: the difference is confined to the spelling of a
  plain stderr label. It cannot alter classification, exit status, artifacts,
  reports, evidence, automation, or Apply authorization. Preserving insertion
  order here would require a new ordered-object runtime solely for inert
  diagnostic punctuation, outside the v2 compatibility boundary.
- Suggested follow-up: retain the explicit allowed-divergence note; revisit
  only if an ordered value becomes a byte-reaching artifact or decision input.
